Two Sum

Problem

Given an array of integers nums and an integer target, return the indices of the two numbers such that they add up to target.

You may assume that each input would have exactly one solution, and you may not use the same element twice.

You can return the answer in any order.

Example:
Input: nums = [2, 7, 11, 15], target = 9
Output: [0, 1]
Explanation: nums[0] + nums[1] = 2 + 7 = 9

Solutions

Suboptimal Solutions

1. Brute Force (Nested Loops)
Iterate through each pair of elements and check if they sum to target.

2. Sorting + Two Pointers
Sort the array, then use two pointers from both ends to find the pair.

Optimal Solution

The key insight is to use a hash map (dictionary) to store numbers we've already seen along with their indices. As we iterate through the array, for each number we check if its complement (target - current number) exists in our hash map.

Why this works: If we're looking at number x and need target, then we need to find target - x. Instead of searching the entire array each time (which would be O(n²)), we maintain a hash map of previously seen numbers for O(1) lookup. This approach trades space for time, reducing the complexity from O(n²) to O(n) by eliminating the need for nested loops.

Algorithmic Pattern: Hash Map (One-Pass)

Algorithm for Optimal Solution

  1. Create an empty hash map to store numbers and their indices
  2. Iterate through the array with index i
  3. For each element nums[i], calculate the complement: complement = target - nums[i]
  4. Check if the complement exists in the hash map:
    • If yes: return [hash_map[complement], i]
    • If no: add nums[i] to hash map with its index
  5. Continue until solution is found

Edge Cases

  • Negative numbers: Algorithm works with negative integers
  • Duplicates: Same value can appear multiple times (use different indices)
  • Zero values: Handle cases where nums[i] = 0 or target = 0
  • Minimum array size: Array must have at least 2 elements
  • Order doesn't matter: Return indices in any order

Complexity

Approach Time Complexity Space Complexity
Brute Force (nested loops) O(n²) O(1)
Sorting + Two Pointers O(n log n) O(1)
Hash Map (Optimal) O(n) O(n)

Optimal Solution Implementation

Java 
                
private static int[] twoSum(int[] nums, int target) {
  Map<Integer, Integer> map = new HashMap<>();
  for (int i = 0; i < nums.length; i++) {
      int complement = target - nums[i];
      if (map.containsKey(complement)) {
          // order does not matter
          return new int[]{map.get(complement), i};
          // return new int[]{i, map.get(complement)};
      }
      map.put(nums[i], i);
  }
  return new int[]{-1, -1};
}
Python
                
def two_sum(nums, target):
    index_map = {}
    for i, num in enumerate(nums):
        complement = target - num
        if complement in index_map:
            # order does not matter
            return [index_map[complement], i]
            # return [i, index_map[complement]]
        index_map[num] = i
    return [-1, -1]
