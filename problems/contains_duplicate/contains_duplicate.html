Contains Duplicate

Problem

Given an integer array nums, return true if any value appears at least twice in the array, and return false if every element is distinct.

Input: Integer array nums.
Output: Boolean indicating if duplicates exist.
Constraints: 1 ≤ nums.length ≤ 10^5.

Example:
Input: nums = [1, 2, 3, 1]
Output: true
Explanation: The value 1 appears at least twice.

Solutions

Suboptimal Solutions

1. Brute Force (Nested Loops)
Compare each pair of elements to check for equality. This is O(n²) time.

2. Sorting + Scan
Sort the array then check adjacent elements. Sorting costs O(n log n) time.

Optimal Solution

Use a hash set to track values we have seen. If we encounter a value already in the set, we found a duplicate.

Why this works: A hash set provides O(1) average-time lookups and insertions, so we can detect duplicates in a single pass.

Algorithmic Pattern: Hash Set (One-Pass)

Algorithm for Optimal Solution

  1. Initialize an empty hash set.
  2. Iterate through each number in nums.
  3. If the number is already in the set:
    • Return true (duplicate found).
    • Otherwise, add the number to the set.
  4. If the loop ends, return false.

Edge Cases

  • All unique values: Must return false.
  • Immediate duplicate: Example [2, 2] should return true quickly.
  • Large input: Ensure linear time with a hash set for 10^5 items.
  • Negative values: Hash set handles negatives without special cases.

Complexity

Approach Time Complexity Space Complexity
Brute Force (nested loops) O(n²) O(1)
Sorting + Scan O(n log n) O(1)
Hash Set (Optimal) O(n) O(n)

Optimal Solution Implementation

Java
import java.util.HashSet;
import java.util.Set;

public class Solution {
    public boolean containsDuplicate(int[] nums) {
        Set<Integer> seen = new HashSet<>();
        for (int num : nums) {
            if (seen.contains(num)) {
                return true;
            }
            seen.add(num);
        }
        return false;
    }
}
